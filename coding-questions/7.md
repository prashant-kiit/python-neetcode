# Sets are unordered but Dists are not. How? In short.
**How?** In short:

* **`dict`** maintains **two structures** internally:

  1. A **hash table** for fast lookups.
  2. A **compact array of entries** that records items in the order they were inserted.

  Iteration walks the entries array, so keys come out in insertion order.

* **`set`** uses **only a hash table**. When you iterate, Python walks the hash table buckets, whose layout depends on hashes, collisions, and resizing—not insertion order.

**One-line interview answer:**

> A `dict` preserves order because it stores entries in insertion order in addition to using a hash table, while a `set` uses only a hash table, so its iteration order is not guaranteed.

Good question. The key is to remember there are **two different kinds of objects** involved:

1. **The class object** (`Foo`)
2. **The instance object** (`f`)

`Meta.__new__` and `Meta.__init__` are used when **creating the class (`Foo`)**, not when creating an instance (`f`).

For example:

```python
class Meta(type):
    def __new__(mcls, name, bases, namespace):
        print("Meta.__new__")
        return super().__new__(mcls, name, bases, namespace)

    def __init__(cls, name, bases, namespace):
        print("Meta.__init__")
        super().__init__(name, bases, namespace)

    def __call__(cls, *args, **kwargs):
        print("Meta.__call__")
        return super().__call__(*args, **kwargs)

class Foo(metaclass=Meta):
    def __new__(cls):
        print("Foo.__new__")
        return super().__new__(cls)

    def __init__(self):
        print("Foo.__init__")

print("Class defined")
f = Foo()
```

### Output

```text
Meta.__new__
Meta.__init__
Class defined
Meta.__call__
Foo.__new__
Foo.__init__
```

## Timeline

### Phase 1: Class creation (runs once)

When Python executes:

```python
class Foo(metaclass=Meta):
    ...
```

it effectively does:

```python
Foo = Meta("Foo", bases, namespace)
```

This calls:

```
Meta.__new__()
        ↓
Meta.__init__()
```

After this, the class object `Foo` exists.

---

### Phase 2: Instance creation (runs every `Foo()`)

Later:

```python
f = Foo()
```

becomes:

```python
Meta.__call__(Foo)
```

which internally does:

```
Foo.__new__()
        ↓
Foo.__init__()
```

---

## Complete lifecycle

```
class Foo(metaclass=Meta):
        │
        ▼
Meta.__new__      ← create the class object
        │
        ▼
Meta.__init__     ← initialize the class object
        │
        ▼
Foo exists
        │
        ▼
Foo()             ← create an instance
        │
        ▼
Meta.__call__
        │
        ▼
Foo.__new__       ← allocate instance
        │
        ▼
Foo.__init__      ← initialize instance
        │
        ▼
instance returned
```

## Simple rule to remember

| Method          | Called when      | Creates/Initializes        |
| --------------- | ---------------- | -------------------------- |
| `Meta.__new__`  | `class Foo(...)` | **Class object** (`Foo`)   |
| `Meta.__init__` | `class Foo(...)` | **Class object** (`Foo`)   |
| `Meta.__call__` | `Foo()`          | Controls instance creation |
| `Foo.__new__`   | `Foo()`          | **Instance** (`f`)         |
| `Foo.__init__`  | `Foo()`          | **Instance** (`f`)         |

A helpful mental model is:

* `Meta.__new__` / `Meta.__init__` build the **blueprint** (the class).
* `Meta.__call__` is the **factory manager** that decides how to make objects from that blueprint.
* `Foo.__new__` / `Foo.__init__` build and initialize each **actual object**.

Here are the core concepts involved:

1. **Metaclasses** — a class whose instances are classes themselves (the "class of a class"). Controls how classes are constructed.

2. **`type` as the default metaclass** — every ordinary class in Python is normally an instance of `type`. Subclassing `type` lets you create a custom metaclass.

3. **`__new__` vs `__init__` in class creation** — `__new__` is responsible for actually *creating* the object (here, a class object) before it's initialized. Overriding it in a metaclass intercepts class construction.

4. **Class creation parameters** — `name`, `bases`, `ns` (namespace):
   - `name`: string name of the class
   - `bases`: tuple of parent classes
   - `ns`: dictionary of the class body's attributes/methods before the class object exists

5. **`metaclass=` keyword argument** — explicitly specifies which metaclass to use when defining a class (`class Base(metaclass=Meta)`).

6. **Metaclass inheritance** — if a base class uses a custom metaclass, subclasses automatically use the same metaclass too, even without explicitly declaring it.

7. **Namespace mutation before class construction** — modifying the `ns` dict inside `__new__` lets you inject attributes/methods into a class *before* it's fully built, so they become part of the class itself (not just inherited).

8. **`super().__new__(...)`** — delegates back to the parent metaclass (`type`) to actually construct the class object once customization is done.

9. **Class-time vs instance-time execution** — the printed messages and attribute injection happen when the *class itself* is defined (at import/module-load time), not when an object is instantiated from the class.

10. **Attribute ownership vs inheritance** — distinguishing whether an attribute exists directly in a class's own `__dict__` versus being looked up via inheritance (MRO) — relevant here since each class gets its own separately injected `created_by`.

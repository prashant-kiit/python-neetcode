`__call__` is a special (dunder) method that makes an **object callable like a function**.

### Basic example

```python
class Adder:
    def __call__(self, x, y):
        return x + y

a = Adder()

print(a(2, 3))      # 5
print(a.__call__(2, 3))  # 5 (same thing)
```

When Python sees:

```python
a(2, 3)
```

it internally does:

```python
a.__call__(2, 3)
```

---

### Why use it?

It lets an object behave like a function while still maintaining state.

```python
class Counter:
    def __init__(self):
        self.count = 0

    def __call__(self):
        self.count += 1
        return self.count

c = Counter()

print(c())  # 1
print(c())  # 2
print(c())  # 3
```

Here, a normal function couldn't easily keep `count` as instance state.

---

### Functions themselves have `__call__`

```python
def greet(name):
    return f"Hello {name}"

print(greet("Alice"))          # Hello Alice
print(greet.__call__("Alice")) # Hello Alice
```

This works because function objects implement `__call__`.

---

### Common use cases

* Functor (function-like object)
* Stateful callbacks
* Decorator classes
* Machine learning models

Example:

```python
class Model:
    def __call__(self, x):
        return x * 2

model = Model()

print(model(10))  # 20
```

This is why libraries like PyTorch let you write:

```python
output = model(input)
```

instead of:

```python
output = model.forward(input)
```

Internally, `model(...)` eventually invokes `model.__call__(...)`.

### Summary

* `obj()` → calls `obj.__call__()`.
* Implement `__call__` when you want instances of a class to behave like functions.
* It is commonly used for stateful function-like objects, decorators, callbacks, and frameworks such as PyTorch.

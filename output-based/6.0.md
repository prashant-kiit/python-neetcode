This code demonstrates **multiple inheritance** and Python's **Method Resolution Order (MRO)** — specifically the classic "diamond problem."

### The class hierarchy
```
      A
     / \
    B   C
     \ /
      D
```
- `A` defines `who()` returning `"A"`.
- `B` inherits from `A`, overrides `who()` to return `"B"`.
- `C` inherits from `A`, overrides `who()` to return `"C"`.
- `D` inherits from **both** `B` and `C`, and defines no methods of its own — this is the "diamond" shape, since both `B` and `C` trace back to the same ancestor `A`.

### `D().who()`
Since `D` has no `who()` method itself, Python looks it up following the **MRO** — the linear order in which base classes are searched for attributes/methods. Because `B` is listed *before* `C` in `class D(B, C)`, Python finds `B`'s `who()` first and uses it — so this returns `"B"`.

### `D.__mro__`
This reveals the actual linearization order Python computed for resolving attributes on `D`. Python doesn't use naive depth-first search (which would cause `A` to be checked before `C`, an inconsistency); instead, it uses the **C3 linearization algorithm**, which guarantees:
1. A class always appears before its parents.
2. The order of base classes as declared (`B` before `C`) is preserved.
3. Each class appears exactly once, even if reachable via multiple paths.

The resulting MRO here is `[D, B, C, A, object]` — `D` first, then `B` and `C` (in the order declared), then their shared ancestor `A` only *once* (not twice, despite two paths reaching it), and finally `object` (Python's universal base class).

### Key concepts involved
1. **Multiple inheritance** — a class inheriting from more than one base class.
2. **The diamond inheritance problem** — ambiguity that arises when two parents share a common ancestor.
3. **Method Resolution Order (MRO)** — the specific, deterministic order Python searches classes when resolving an attribute or method.
4. **C3 linearization** — the algorithm Python uses to compute the MRO, ensuring consistency and avoiding duplicate/contradictory ordering.
5. **`__mro__`** — a built-in tuple attribute on every class showing its computed resolution order, useful for debugging inheritance behavior.

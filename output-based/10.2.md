This example demonstrates the **complete object creation process** in Python.

```python
class Meta(type):
    def __call__(cls, *args, **kwargs):
        print("Meta.__call__")
        instance = super().__call__(*args, **kwargs)
        print("instance created")
        return instance

class Foo(metaclass=Meta):
    def __new__(cls):
        print("Foo.__new__")
        return super().__new__(cls)

    def __init__(self):
        print("Foo.__init__")

f = Foo()
```

## Step 1: `Foo()` is called

```python
f = Foo()
```

Since `Foo` is **an instance of `Meta`**, Python doesn't directly create the object.

Instead it does:

```python
Meta.__call__(Foo)
```

So the first output is

```
Meta.__call__
```

At this point:

* `cls` = `Foo`

---

## Step 2: `super().__call__()`

```python
instance = super().__call__(*args, **kwargs)
```

`super()` refers to `type`, so this is effectively:

```python
type.__call__(Foo)
```

Now `type.__call__` performs the standard object creation process.

Internally it is approximately:

```python
obj = Foo.__new__(Foo)

if isinstance(obj, Foo):
    Foo.__init__(obj)

return obj
```

---

## Step 3: `Foo.__new__`

```
Foo.__new__
```

This allocates memory and returns a new `Foo` object.

Equivalent to

```python
obj = object.__new__(Foo)
```

---

## Step 4: `Foo.__init__`

```
Foo.__init__
```

This initializes the already-created object.

---

## Step 5: Back to `Meta.__call__`

Now

```python
instance = super().__call__(*args, **kwargs)
```

has returned the initialized object.

Execution continues:

```python
print("instance created")
return instance
```

Output:

```
instance created
```

---

## Final output

```
Meta.__call__
Foo.__new__
Foo.__init__
instance created
```

# Why use a metaclass `__call__`?

Because it lets you intercept **every object creation**.

For example, Singleton:

```python
class Singleton(type):
    _instance = None

    def __call__(cls, *args, **kwargs):
        if cls._instance is None:
            cls._instance = super().__call__(*args, **kwargs)
        return cls._instance
```

Now every

```python
Foo()
```

passes through the metaclass before any instance is created.

---

## One-line summary

* `Foo()` **does not directly call** `Foo.__new__` or `Foo.__init__`.
* It first calls `Meta.__call__` because `Foo` is an instance of the metaclass `Meta`.
* `Meta.__call__` delegates to `type.__call__`, which executes `Foo.__new__` → `Foo.__init__`, returns the instance, and then control returns to `Meta.__call__`.

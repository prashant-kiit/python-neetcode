Here's how to define, raise, and catch custom exceptions in Python.

### 1. Define a custom exception
Custom exceptions are just classes that inherit from `Exception` (or a more specific built-in exception).

```python
class InsufficientBalanceError(Exception):
    """Raised when an account doesn't have enough balance for a withdrawal."""
    def __init__(self, balance, amount):
        self.balance = balance
        self.amount = amount
        super().__init__(f"Cannot withdraw {amount}, balance is only {balance}")
```

### 2. Throw (raise) it
```python
def withdraw(balance, amount):
    if amount > balance:
        raise InsufficientBalanceError(balance, amount)
    return balance - amount
```

### 3. Catch it
```python
try:
    withdraw(100, 150)
except InsufficientBalanceError as e:
    print("Error:", e)
    print("Balance was:", e.balance)
    print("Attempted amount:", e.amount)
```

Output:
```
Error: Cannot withdraw 150, balance is only 100
Balance was: 100
Attempted amount: 150
```

### Catching multiple exception types
```python
try:
    withdraw(100, 150)
except InsufficientBalanceError as e:
    print("Balance issue:", e)
except ValueError as e:
    print("Value issue:", e)
except Exception as e:
    print("Something else went wrong:", e)
```

### Custom exception hierarchy
You can build a family of related exceptions by inheriting from a common base:

```python
class BankError(Exception):
    """Base class for all bank-related errors."""
    pass

class InsufficientBalanceError(BankError):
    pass

class InvalidAccountError(BankError):
    pass

try:
    raise InvalidAccountError("Account not found")
except BankError as e:   # catches InsufficientBalanceError, InvalidAccountError, or any subclass
    print("Bank error occurred:", e)
```

### Re-raising an exception
Sometimes you want to log/handle it partially and still propagate it:

```python
try:
    withdraw(100, 150)
except InsufficientBalanceError as e:
    print("Logging error:", e)
    raise  # re-raises the same exception
```

### `finally` (runs no matter what)
```python
try:
    withdraw(100, 150)
except InsufficientBalanceError as e:
    print("Error:", e)
finally:
    print("Transaction attempt finished.")
```

### Key points
- Always inherit from `Exception` (avoid inheriting from `BaseException` directly — that's reserved for things like `SystemExit`/`KeyboardInterrupt`).
- Use `super().__init__(message)` so the exception has a proper string message.
- Catch specific exceptions before generic ones (`except Exception` should usually go last).
- Building an exception hierarchy (like `BankError` above) lets you catch broad or narrow categories as needed.
